Quick Steps

1. Resource Hacker - Add Icons

  1. Open Resource Hacker
  2. Open gfx023.egf (select "All Files" to see it)
  3. Scroll to bottom, find last numbers (e.g., 1047, 1048)
  4. Actions > Add (Ctrl+M), add your BMPs as next numbers (1049, 1050)
  5. Save

2. Resource Hacker - Add Armor Graphics

  1. Open armor GFX file (e.g., gfx0014.egf for female armor)
  2. Calculate position: If last sprite ends at 2622, add 29 gap = start at 2651, end 2672
  3. Add 22 armor sprites in that range
  4. Save

3. PubEditor Configuration

  1. Open PubEditor, press plus icon
  2. Name your item
  3. Click "Last" (inventory graphic displays)
  4. Change item type from "General" to "Armour"
  5. Look to your right, select Armour Resource ID, scroll down to find associated animation graphic
  6. Set GRD size = 2by3, graphics IDs (1049, 1050)
  7. Alter weights etc if you want
  8. Press "Create"
  9. Add EOF entry at end (empty item)
  10. Save

4. Deploy to Server

  1. Move your updated files into your server emulator (e.g., REOSERV)
  2. To spawn your item, use the index number (e.g., 484) with admin command: $si 484
  3. Your custom item now works in-game!

Key Points

  • Icons: Drop (odd number), Inventory (even number)
  • Armor gap: Always 29 sprites between sets
  • GRD size: 2by3 for armor items
  • EOF required: Always add empty item at end
  • Format: BMP 24-bit

Troubleshooting

  • Can't see EGF: Select "All Files" in Resource Hacker
  • Wrong sprite: Check sprite IDs match between Resource Hacker and PubEditor
  • Client crash: Missing EOF entry or corrupt BMP
